# Codex 防误发

一个只针对 Codex 桌面 App 的 macOS 菜单栏工具。它会在 Codex 位于前台时，把普通回车键改成换行；保留 `Command + Return` 作为发送。

适合这样的使用习惯：平时写提示词时随手按回车只想换行，真正发送时再按 `Command + Return`。

## 效果

- Codex 里按普通回车：换行。
- Codex 里按 `Command + Return`：保持 Codex 原生发送行为。
- 其他 App 里按回车：不做任何改写。

## 下载安装

1. 在仓库的 `dist/` 目录里下载 `Codex防误发.app.zip`。
2. 解压后得到 `CodexSendGuard.app`。
3. 建议把它拖到“应用程序”文件夹。
4. 双击启动后，菜单栏会出现 `⌘↵`。
5. 第一次启动时，根据状态窗口提示授予权限。

## 权限

macOS 会要求授予两个权限：

- 辅助功能
- 输入监控

原因是这个工具需要在系统层面监听并改写键盘事件。工具不会修改 Codex 本体，也不会联网。

如果重新编译或重新签名 App，macOS 可能会把它当成一个新的程序，需要重新授予权限。

## 使用

启动后，菜单栏会显示 `⌘↵`。保持它在后台运行即可。

菜单里包含：

- 暂停 / 启用
- 显示状态窗口
- 键盘监听状态
- 请求权限
- 开机启动
- 退出

## 开机启动

点击菜单栏的 `⌘↵`，选择“开机启动”。状态变为“开机启动：已开启”后，下次登录 macOS 时会自动启动。

## 常见问题

### 菜单栏看不到 `⌘↵`

先确认 App 已经启动。如果仍然看不到，可以再次双击 App，或者在“活动监视器”里搜索 `CodexSendGuard`。

### 状态窗口一直显示需要授权

进入“系统设置”：

1. 打开“隐私与安全性”。
2. 进入“辅助功能”，打开 `Codex 防误发` 或 `CodexSendGuard`。
3. 进入“输入监控”，打开 `Codex 防误发` 或 `CodexSendGuard`。
4. 退出并重新打开 App。

如果列表里有旧名字或重复项，可以先删除旧项，再重新添加当前 App。

### 授权后还是不能拦截回车

确认状态窗口里：

- 辅助功能：已开启
- 输入监控：已开启
- 键盘监听：开启

然后把 Codex 切到前台再测试。这个工具只在 Codex 前台时生效。

### 更新版本后又要重新授权

这是 macOS 的权限机制导致的。App 重新签名、重新编译或移动到不同位置后，系统可能会把它识别成新的程序，需要重新授权。

## 构建

```bash
swift test

```

## 打包

最终 App 包位于：

```text
outputs/CodexSendGuard.app
```

源码里的打包信息位于：

```text
work/CodexEnterGuard/Packaging/Info.plist
```

其中用户可见名称是 `Codex 防误发`，Bundle Identifier 是 `local.codex-send-guard`，可执行文件名是 `CodexSendGuard`。

## 隐私说明

这个工具只在本机运行，不联网，不上传数据，不读取 Codex 内容。它只检查当前前台 App 是否是 Codex，并在满足条件时把普通回车事件改成带 Shift 的回车事件。
